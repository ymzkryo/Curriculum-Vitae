## 保有スキル

- RESTful API の開発： Python（Flask、Django、FastAPI）や Ruby on Rails を用いた高効率な RESTful API の設計・実装に豊富な経験。数万件/秒のリクエストに耐えるスケーラブルな API を構築
- AWS を活用したインフラ構築： EC2、S3、Lambda、RDS、CloudFront、API Gateway など AWS の主要サービスを駆使し、セキュアかつスケーラブルなクラウド環境を構築
- CI/CD パイプラインの導入： GitHub Actions や CircleCI を使用した CI/CD 環境の構築により、デプロイ時間を 30％短縮し、開発効率を大幅に向上
- Docker によるコンテナ化： Docker および Docker Compose を使用して、開発環境の統一と本番環境のスムーズなデプロイを実現
- Git/Github を使ったチーム開発： チームでのバージョン管理を適切に行い、レビュー・プルリクエストの運用によりコード品質を向上

## 得意分野

- バックエンドからインフラ構築まで： RESTful API の設計・実装から、AWS を活用したインフラの構築まで一貫して対応可能。パフォーマンス最適化やセキュリティ対策に注力
- 要求分析から運用保守まで： 要件定義・設計フェーズから、テスト・運用保守に至るまでの全フェーズに関与し、プロジェクト全体を俯瞰してマネジメント
- パフォーマンスチューニング： 特に API のパフォーマンス最適化に強みを持ち、データベースやサーバーサイドの負荷分散、キャッシュの導入による処理速度の向上を実現

## 自己PR

- 効率的なドキュメント： チームやクライアント向けに分かりやすい技術ドキュメントや設計書を作成することが得意。ドキュメント自動生成ツールの導入により、工数削減にも貢献
- 自動化の追求： CI/CD パイプラインの構築や、繰り返しの作業を自動化することで、チーム全体の生産性を向上させることを目指しています。業務フローの最適化やスクリプト作成により、運用コストを削減
- チームプレイヤーとしての貢献： チーム開発において積極的にコミュニケーションを取り、問題解決やリスク回避に貢献。コードレビューやペアプログラミングを通じて、他のメンバーと共にコード品質を向上させる取り組みをしています

### 案件名

- 紡績機械の生産統計閲覧アプリの開発

### 期間

- 2016 年 11 月 ~ 2018 年 12 月

### 担当業務

- AWS インフラ構築とパフォーマンス最適化
- 生産統計データ取得機能の設計・開発
- ニュース配信機能の開発

### 役割・規模

- チーム人数：3 人
- インフラ設計および統計データ取得システムの設計・実装を担当



### 言語・ツール・インフラ

- Java
  - Spring
- Python
- AWS
  - CloudFront
  - Lambda
  - EC2(AmazonLinux, WindowsServer)
  - RDS(Oracle)

### 主な成果

- CloudFront を導入し、世界中のユーザーに対して 2 秒以内のレスポンスを実現し、パフォーマンスを大幅に改善
- 生産統計データの取得システムをポーリング処理に変更することで、安定したデータ処理と効率的なデータ収集を実現
- 生産統計データが 5 分間に 1 回生成される仕様に基づき、設計段階では AWS Lambda による並列処理を検討したが、Lambda のタイムアウト制限によりサーバ側でポーリング処理に切り替え、安定したデータ処理を実現
- ニュース配信機能を追加し、ユーザーにリアルタイムで情報を提供するシステムを構築
