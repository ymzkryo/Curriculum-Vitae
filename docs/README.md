### 案件名

- 紡績機械の生産統計閲覧アプリの開発案件

### 期間

- 2016 年 11 月~2018 年 12 月

### 担当業務

- AWS インフラ構築
- 生産統計の取得機能開発
- ニュース機能開発

### 役割・規模

- チーム人数： 3 人

### コメント

世界中のどこからアクセスされても 2 秒以内にレスポンスを返す必要があったため、  
キャッシュサーバ（CloudFront)を前段に配置して、レスポンスを担保した。  
生産統計データは SQL の生ファイルで 5 分間に 1 回以上のペースで生成される仕様だったので  
設計段階では Lambda で並列処理による DB への流し込みを考えていたが、  
Lambda のタイムアウトと DB への流し込み時間が合わなかったため、サーバでのポーリング処理に  
置き換えた。
